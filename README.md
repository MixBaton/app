# app

1. Введение
1.1. Цель
 
 Разработать веб-приложение "Доска объявлений" для удобного размещения, просмотра и управления объявлениями.
 Основные задачи:

•	Предоставить пользователям интуитивно понятный интерфейс для работы с объявлениями

•	Реализовать функционал избранных объявлений с сохранением состояния

•	Обеспечить кросс-платформенную совместимость и адаптивный дизайн

•	Создать современный пользовательский интерфейс 

1.2. Область применения
Приложение предназначено для:

•	Частных лиц, желающих размещать объявления о продаже товаров или услуг

•	Пользователей, ищущих товары или услуги 

•	Малого бизнеса для продвижения своих предложений

Основные пользователи: физические лица с базовыми навыками работы с веб-браузерами.

1.3. Объем проекта

Входит в проект:

•	Фронтенд-приложение на HTML/CSS

•	Система управления объявлениями (просмотр, добавление в избранное)

•	Локальное хранение данных через LocalStorage

•	Адаптивный пользовательский интерфейс

Не входит в проект:

•	Система аутентификации пользователей

•	Модерация объявлений

•	Система платежей и транзакций


1.4. Определения, аббревиатуры и сокращения

•	UI (User Interface) - пользовательский интерфейс

•	UX (User Experience) - пользовательский опыт

•	SRS (Software Requirements Specification) - спецификация требований к программному обеспечению

•	LocalStorage - веб-хранилище браузера для сохранения данных на стороне клиента

•	CSS - каскадные таблицы стилей

•	API - интерфейс программирования приложений

•	CSS Variables: https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties

2. Общее описание
 
 2.1. Перспективы продукта

Приложение решает проблему организации и удобного доступа к локальным объявлениям. Оно вписывается в экосистему веб-приложений как легковесное клиентское решение, не требующее серверной инфраструктуры.

2.2. Функции продукта

•	Просмотр списка объявлений с фильтрацией по категориям

•	Добавление объявлений в избранное

•	Управление списком избранных объявлений

•	Адаптивное отображение на различных устройствах
•
Локальное сохранение состояния избранных объявлений


2.3. Характеристики пользователей
Целевая аудитория:

•	Возраст: 18-65 лет

•	Уровень технической подготовки: базовый (умение пользоваться браузером)

•	Потребности: удобная система для работы с объявлениями


2.4. Ограничения

•	Платформа: веб-браузеры, поддерживающие HTML5 и CSS3

•	Сроки разработки: 2-3 недели

•	Бюджет: открытый исходный код

•	Технологии: чистый JavaScript без фреймворков

2.5. Предположения и зависимости
 
 Предположения:

•	Пользователи имеют современные браузеры с поддержкой LocalStorage

•	Объявления загружаются из предопределенного источника

Зависимости:

•	Bootstrap 5.1.3 для компонентов интерфейса

•	Font Awesome 6.0.0 для иконок

•	Современные браузеры с поддержкой ES6+


3. Функциональные требования


3.1. Требования к интерфейсу пользователя

Система должна отображать список объявлений в виде карточек

•	Каждая карточка должна содержать: заголовок, описание, цену, категорию
•	Карточки должны иметь визуальное разделение и тени

 Пользователь должен иметь возможность добавлять объявления в избранное

•	Кнопка добавления в избранное должна быть доступна на каждой карточке
•	Состояние избранного должно сохраняться между сеансами

Система должна предоставлять отдельную страницу для просмотра избранных объявлений

•	На странице должен отображаться счетчик избранных элементов
•	Должна быть возможность удаления объявлений из избранного

Интерфейс должен быть адаптивным и корректно отображаться на мобильных устройствах

•	Минимальная ширина экрана: 320px
•	Максимальная ширина экрана: 3840px


3.2. Требования к данным

Система должна хранить данные избранных объявлений в LocalStorage

•	Данные должны сохраняться в формате JSON
•	При загрузке страницы данные должны восстанавливаться

Система должна обрабатывать отсутствие объявлений в избранном
•	Должно отображаться состояние "пустой список" с соответствующим сообщением


4. Нефункциональные требования

4.1. Производительность

Время загрузки страницы не должно превышать 3 секунд на стандартном подключении

4.2. Надежность

Приложение должно работать без сбоев в современных браузерах
Данные избранного должны сохраняться корректно при обновлении страницы

4.3. Удобство использования
Интерфейс должен быть интуитивно понятен для пользователей с базовыми компьютерными навыками
Навигация между страницами должна быть простой и очевидной

4.4. Совместимость
Приложение должно корректно работать в:

•	Chrome 90+

•	Firefox 88+

•	Safari 14+

•	Edge 90+



5. Внешний интерфейс


5.1. Интерфейс пользователя (UI)

Навигационная панель:

•	Логотип и название приложения

•	Ссылка на главную страницу с объявлениями

•	Ссылка на страницу избранного с счетчиком

Карточка объявления:

•	Заголовок с категорией

•	Цена выделенным шрифтом

•	Описание товара/услуги

•	Кнопка управления избранным

Страница избранного:

•	Заголовок страницы

•	Сетка карточек избранных объявлений

•	Кнопка удаления из избранного

•	Состояние пустого списка

5.2. Интерфейс программирования приложений (API)

Локальное хранилище:


•	Формат данных: массив объектов объявлений

•	Структура объекта: {id, title, description, price, category}
